学习犹如逆水行舟,不进则退。

亲爱的读者你好，首先我先声明一下，这并不是一篇技术性很强的文章，你看了这篇文章，也不会几分钟就能掌握Kotlin（其实也不可能）。我给这篇文章的定位是：如果你看完这篇文章后，就迫不及待的想尝试在你的项目中使用Kotlin，那么这篇文章就成功了~

Kotlin的前世今生

Kotlin的作者是大名鼎鼎的Jetbrains公司。它有一系列耳熟能详的产品，诸如Android开发兄弟们天天用的Android Studio，还有比较有名的IntelliJ IDEA，WebStorm，PhpStorm等。

其实Kotlin在2015年都已经有人在使用了，只不过因为在2017年Google I/O大会上被官方推荐为Android的官方语言之一后，成为了大热。目前国内一些有名的公司也都在推荐使用Kotlin开发，前段时间看到国外开发者论坛上说Google负责Android安全的团队已经将Kotlin作为主力开发语言了，相信未来你再看Android源码的时候就可能是Kotlin编写的了。

Kotlin好在哪里呢

听我扯了半天的皮，下面我来列举一些Kotlin在日常开发中一些实例，让你知道为什么Kotlin让Android开发更优雅更效率。

和Java无缝连接

其实在项目组引入一门新语言，最害怕的是和现开发语言出现兼容问题，这也是一些小伙伴不敢轻易尝试的原因。Ok，经过一段时间的使用和一些论坛的讨论，我发现他们真的兼容的很无缝！无缝到让你感觉不到他们是不同的语言（除了语法不一样）！官方点说就是他们都是基于JVM的一种高级语言，最后编译后都变成了字节码，Kotlin 的标准库更多的是对 Java 库的扩展，基于这个设计思路，你丝毫不需要担心 Kotlin 对 Java 代码的引用，你甚至可以在 Kotlin 当中使用 Java 反射，反正只要是 Java 有的，Kotlin 都有。形象点说，就像一个人（程序）穿了两件不同颜色的衣服（Java、Kotlin），不管怎么来回换衣服，最后脱了衣服（编译好）都还是同一个人。

Kotlin 的标准库更多的是对 Java 库的扩展，基于这个设计思路，你丝毫不需要担心 Kotlin 对 Java 代码的引用，你甚至可以在 Kotlin 当中使用 Java 反射，反正只要是 Java 有的，Kotlin 都有，它们可以像调用自己的函数一样调用对方的，无论是第三方库，还是热更新，SDK云云都可以无任何兼容问题。

学习门槛极低

就我个人经历来说，我当时是下午快下班的时候看了两个小时左右的Kotlin语法，第二天就可以大致使用了（当然遇到不会的还是查阅相关资料，哈哈）。其实就我个人感觉而言，其实Kotlin的语法有点像Python（IOS小伙伴说Kotlin像Swift），因为当时回家我也在学Python，就觉得它们的语法真的好相似，当时我怕学混了，还特意把Python的进度放慢了，嘿嘿，原谅我很笨，没法同时记。

其实，作为一名Android开发工程师，学Kotlin语法就更简单了，因为Android Studio直接可以将Java转为Kotlin，不光支持整个文件转换（不推荐大文件，会有好多报错，review费时还易出错），而且还可以直接复制粘贴进行转换，如果你有哪里语法搞不明白了，直接粘贴过去一对比就学会了，是不是很方便呢？那么问题来了，为什么编译器会支持的这么到位？别问我为什么，请往前翻去看《Kotlin的前世今生》。

空指针异常

Null是Java中最常见的陷阱之一，在很多时候不得不写非空判断来增加代码的安全性，随着业务的增长，非空的判断会越来越多，难免会失去控制，也会让代码变得非常的臃肿难懂，而Kotlin的一大特色就是空安全，kotlin定义变量时区分两种类型：

  • var mNullable: Any? = null 可空
  • var mNonNull: Any = XXX 非空

当mNullable变量未判空时，编译是无法通过的，也就是说所有为Null的变量都是你允许的，这就杜绝了很多的遗漏的空指针异常。除此之外，Kotlin的判空语句也是非常简洁的：

总结

最后借用Kotlin的官方文档的介绍来当本文的总结：

  • 兼容性：Kotlin 与 JDK 6 完全兼容，保障了 Kotlin 应用程序可以在较旧的 Android 设备上运行而无任何问题。Kotlin 工具在 Android Studio 中会完全支持，并且兼容 Android 构建系统。
  • 性能：由于非常相似的字节码结构，Kotlin 应用程序的运行速度与 Java 类似。 随着 Kotlin 对内联函数的支持，使用 lambda 表达式的代码通常比用 Java 写的代码运行得更快。
  • 互操作性：Kotlin 可与 Java 进行 100％ 的互操作，允许在 Kotlin 应用程序中使用所有现有的 Android 库 。这包括注解处理，所以数据绑定和 Dagger 也是一样。
  • 占用：Kotlin 具有非常紧凑的运行时库，可以通过使用 ProGuard 进一步减少。 在实际应用程序中，Kotlin 运行时只增加几百个方法以及 .apk 文件不到 100K 大小。
  • 编译时长：Kotlin 支持高效的增量编译，所以对于清理构建会有额外的开销，增量构建通常与 Java 一样快或者更快。
  • 学习曲线：对于 Java 开发人员，Kotlin 入门很容易。包含在 Kotlin 插件中的自动 Java 到 Kotlin 的转换器有助于迈出第一步。

当然在生产环境中，稳定高于一切仍然是不二法则。追求新技术，一方面会给团队带来开发和维护上的学习成本，另一方面也要承担未来某些情况下因为对新技术不熟悉而产生未知问题的风险，也正是知道这些，我在使用之前也私下做了很多的调研和求证，最终才决定使用它在生产环境的。

作为公司里第一个吃螃蟹的人（赶在宏杰之前，哈哈，某人很生气~），在使用Kotlin的一个月里，我已经被它的便捷和优雅所征服，在生产环境中使用新语言开发并没有拖慢我的工作进度，反而更简洁的代码，让我更专心于重要业务逻辑，这也是我决定将Kotlin推荐给大家的重要原因，希望能给大家的开发带来效率上的提升和帮助。
